# Git Workflow (Local-First)

Goal: keep the repo recruiter-friendly and reproducible without committing large artifacts.

## Branching

- `main`: stable, documented baselines + clean results table.
- `exp/<short-name>`: experiments (new model variants, ablations, risky refactors).

## What We Commit

- code in `src/` and `scripts/`
- docs in `docs/`
- small summaries in `reports/` (tables, run registry, conclusions)

## What We Do Not Commit

- `data/` (datasets)
- `models/` (checkpoints)
- `runs/` (logs + per-epoch JSON)

Those are local artifacts and are regenerated by running the scripts.

## Milestone Commits

We commit when:
- a baseline run is stable (script + README command + results row)
- an improvement is validated (best val ROC-AUC improved)
- we add a new reproducibility feature (standard output paths, JSON schema, etc.)
